# Guide Portal App

## 项目简介

`guide-portal-app` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，旨在提供一个通用、轻量的页面集合，方便索引、维护与分发。

## 目录结构

```
guide-portal-app/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 静态资源（CSS、JS、图片等）
├── README.md       # 本文件
└── index.html      # 入口页面（可选）
```

- `pages/`：每个子目录或文件代表一个独立归档页面，命名清晰，便于检索。
- `assets/`：公共资源文件，可由多个页面引用。
- `index.html`：可作为仓库的主入口，提供页面导航或说明。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，每个页面可以包含完整的 HTML 结构、样式和脚本。页面之间互不依赖，可直接在浏览器中打开使用。

归档页面遵循以下约定：

- 页面文件统一放置在 `pages/` 目录下。
- 文件名应反映页面内容（例如 `example-page.html`），避免使用无意义名称。
- 页面内部可引用 `assets/` 中的资源，路径建议使用相对路径。

## 维护说明

- 新增页面：在 `pages/` 下创建新的 HTML 文件，并确保所有资源引用正确。
- 更新页面：直接修改对应文件，注意保持向后兼容。
- 删除页面：移除对应文件，并检查是否有其他页面或导航链接引用。
- 资源管理：公共资源放入 `assets/`，避免重复存储。

如需要添加或修改页面结构，请遵循已有目录规范，确保仓库整洁可维护。
